# Copilot Prompt Optimization Design

**Date:** 2026-04-02
**Status:** Draft

## Problem

The JiraMaster prompt generation page has three issues:

1. **Epics for non-Jira-worthy items** — The aggressiveness levels control extraction volume but don't filter for whether something justifies structured Jira tracking. The LLM creates epics for trivial tasks, vague ideas, and low-priority items.

2. **Not optimized for Teams Copilot** — The prompt was designed for generic LLM use. When used with Copilot in Teams, it returns markdown-formatted YAML (code fences) instead of a downloadable file. The user must manually ask Copilot to re-provide it as a file. The UI instructions reference "GitHub Copilot Chat" which is a different product.

3. **Copy button silently fails** — The clipboard API requires a secure context. When the app is accessed via LAN IP instead of localhost, `navigator.clipboard.writeText()` fails silently with no user feedback.

## Design

### Workstream 1: Base Prompt Rewrite

**Rewrite `AGGRESSIVENESS_MAP`** from extraction-focused to Jira-centric:

| Level | Label | Prompt Instruction |
|-------|-------|-------------------|
| 1 | Conservative | "Only create epics for explicitly agreed initiatives with clear ownership and multi-story scope. A single task, vague idea, or minor follow-up is NOT an epic — skip it." |
| 2 | Standard | "Create epics for committed work items that need structured tracking. Each epic must justify at least 2 stories. Skip discussion points, parking-lot items, and tasks someone can do in a day." |
| 3 | Aggressive | "Create epics for every identifiable stream of work, including tentative items. Still require that each epic has enough substance for multiple stories — don't create single-story epics." |

**Rewrite `data/prompt_template.txt`:**
- Frame the LLM role around Jira-worthiness: "Only create epics for work that justifies structured multi-story tracking in Jira."
- Add explicit anti-patterns: "Do NOT create an epic for a single task, a minor follow-up, or something one person can complete in a day."
- Reinforce output format for Teams compatibility: "Do NOT wrap output in markdown code fences (no \`\`\`). Do NOT add any text before or after the YAML. Output ONLY raw YAML starting with `epics:`."
- Add `{{COPILOT_MODE_INSTRUCTIONS}}` placeholder between the tuning block and the meeting notes section.

**Update aggressiveness slider labels** in the template JS to match the new Jira-centric descriptions.

**Files:** `prompt_builder.py`, `data/prompt_template.txt`, `templates/prompt/index.html`

### Workstream 2: Copilot-in-Teams Platform Modes

**New "Copilot Mode" dropdown** at the top of the Prompt Controls sidebar:

| Mode | When to Use | Prompt Behavior |
|------|-------------|-----------------|
| In-Meeting | During a live Teams meeting with Copilot | Instructs Copilot to analyze the meeting transcript directly. Replaces the `MEETING NOTES:` section with a transcript analysis instruction (no paste area needed — Copilot has the transcript). Explicitly requests downloadable .yaml file output. |
| Post-Recap | After the meeting, with Copilot's recap | Keeps the `MEETING NOTES:` section for the user to paste the recap. Adds context that the input is a Teams Copilot meeting recap. Explicitly requests downloadable .yaml file output. |

**In-Meeting prompt appendix:**
```
COPILOT INSTRUCTIONS:
You have access to this meeting's transcript. Analyze it directly.
Focus on: decisions made, action items assigned, and deliverables committed to.
Provide your complete response as a single downloadable .yaml file attachment.
Do NOT format it as inline text, code blocks, or chat messages.
```

**Post-Recap prompt appendix:**
```
COPILOT INSTRUCTIONS:
The text below is a meeting recap generated by Microsoft Teams Copilot.
Convert it into the YAML structure specified above.
Provide your complete response as a single downloadable .yaml file attachment.
Do NOT format it as inline text, code blocks, or chat messages.
```

**Dynamic "How to use" instructions** that change based on selected mode:

- **In-Meeting:** Copy prompt -> Paste in Teams meeting Copilot -> Copilot analyzes transcript -> Save .yaml file -> Upload on Import page
- **Post-Recap:** Copy meeting recap from Teams -> Generate prompt -> Paste recap where indicated -> Send to Copilot -> Save .yaml file -> Upload on Import page

**Implementation in `prompt_builder.py`:**
- Add `COPILOT_MODE_MAP` dictionary mapping mode names to instruction blocks
- Update `build_prompt()` to accept `copilot_mode` parameter
- Replace `{{COPILOT_MODE_INSTRUCTIONS}}` with the selected mode's instruction block
- For In-Meeting mode, replace `{{MEETING_NOTES}}` with transcript analysis instruction

**Files:** `prompt_builder.py`, `data/prompt_template.txt`, `templates/prompt/index.html`, `routes/prompt.py`

### Workstream 3: Copy Button Fix

**Problem:** The Copy button at `templates/prompt/index.html:90` calls `copyToClipboard('generatedPrompt')` from `static/app.js`. The function uses `navigator.clipboard.writeText()` which requires HTTPS or localhost. When accessed via LAN IP, it fails silently. The button is also nested inside a `<form>` element.

**Fix:**
1. Move the Copy and Download buttons outside the `<form>` element into the card header independently
2. Add visible error feedback in `copyToClipboard()`: when both clipboard API and fallback fail, show a warning message (e.g., "Copy failed — select the text and use Ctrl+C")
3. Use the button's text/class to show error state (red "Copy failed!" for 3 seconds)

**Files:** `templates/prompt/index.html`, `static/app.js`

## Implementation Order
